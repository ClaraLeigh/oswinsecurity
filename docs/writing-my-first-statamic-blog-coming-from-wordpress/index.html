Starter's Creek

My first Statamic blog after growing up with WordPress

Embarking on a new journey from WordPress to Statamic, I share my first impressions, the smooth installation, the unexpected hurdles, and the features that won me over. Join me as I explore the strengths and quirks of Statamic and set the stage for future discussions on security and web development.

Clara Leigh
September 29th, 2024

Hello friend, welcome to my first ever statamic blog.

Installing statamic was a breeze, I installed the global composer package and started the install process. First up, it asked about a starter kit, to which I naturally said yes. After a quick search, I landed on Starter's Creek. It seemed simple and a little quirky, which is just my style.

Next up, it asked about installing addons, and I immediately chose the static site generator. I figure a security focused blog should probably be static html and cheap to host, which I should be able to do with GitHub Pages and Cloudflare.

Everything wrapped up quickly after a few more basic questions, like whether I wanted a super user etc.

Now I was ready to jump in!

Logging in was next and immediately I notice something not quite right in the main menu.

The duplicate IDs menu item looked wrong, clearly an icon is missing and the alt text was overlaying other text making it hard to read.

Okay, okay, one bug so far, thats not too bad but I did think that this would be perfect out of the box from how it was talked about.

Next up its time to write a blog, so I think of a cool title, I look up some banner images and go to insert it.
Whoops, "This field is required", pops up after uploading a file.

What field you ask? Well lets open the developer tools, go to the network tab and look for an error. There is a 422 error on the assets endpoint, with the following response:

```json
{
    "message": "The given data was invalid.",
    "errors": {
        "folder": [
            "This field is required."
        ]
    }
}
```

It appears that a folder is required, which is odd because I wasnâ€™t prompted to select one, nor was there an option available.

Okay, I open the asset manager in the left menu and then I can see all the files, including a folder button. I even managed to upload the banner image here, as I'm sure I'll return to that blog later.

Now I re-open the blog creation page.... and I can't see the folder or the image I uploaded. Something is still wrong and image upload still gets the same 422 error.

Back to assets, let's look around, I see a "Create Container" button. Clicking on it, it seems to be a page where I can store images and choose a filesystem, etc. Okay, this must be it, lets create one of these bad bois.

I go back into create my blog and try again, but I see no collection when I open browse, just the same few images that come default with the starter kit.

Feeling a bit frustrated, I decided to try uploading the image directly to the main folder within Assets.

Now let's review the experience so far.

What was better than WordPress?

  • The framework, it's built on Laravel, which instantly wins me over

  • The install, I like the terminal install, very dev friendly

  • The community, I have met many a Statamic advocate and all of them are the friendliest people in the world

  • It supports a static site! Hell yes, no vulnerabilities in the framework for me

  • I can host for free?!?! Yep, all I need is GitHub and Cloudflare

Areas for Improvement in Statamic

  • First impressions. WordPress has this beat by far.

  • Bugs, there was so many just getting started that I nearly gave up

  • UX, it feels like the early days in WP, like it was built by devs for devs, at least in the backend

  • The install, it needs the 5min UI install that made WP famous

  • Less is more, there is slightly too many things that I feel the need to learn, which WordPress hides by default. I'm sure it will come with time but at a first look, there are things everywhere that don't have easy explanations. I'm sure I'm about to spend a lot of time in the documentation, something I enjoy but others might not

Where to from here?

Well I am going to start writing some security blogs. This was just my first piece out the door and I used it as a learning experience. I'll likely write about my time studying for the OSCP and OSEP, why I got into security, the first time I got in trouble for hacking, and the one time I pulled a 72hr shift at work because of a security issue.

Wait, what about the WordPress drama, can you post some tea?

I started using WordPress back in high school, I actually got my first job in industry using it. I built three or more WordPress sites for the local environmental council, and was even paid a little for it! Paid working for a non profit helping the environment, it was a really sweet first job. Here is what the site looked like, please don't judge me ðŸ™ˆ
It was meant to be a tree with wood carved out for the posts. There was a banner image too and the pacing was slightly better but the way back machine has a pretty close copy of what it looked like.

At the time, I looked in on the WordPress core and saw a bunch of really nice friendly devs and the leader, Matt Mullenweg. He seemed quite excited that people were using WP and wanted to make it grow. He was a little controlling and made some poor design decisions security wise so I kept my distance. I didn't really want to have to fight for my ideas like I saw some doing.

My next major open source platform I used was OpenCart, and the guy running that was about 10x worse. This confirmed my bias that working in open source was essentially inviting drama into my life. Something I didn't really want, so naturally I stayed clear (I was wrong but more on that later).

Following WordPress over the years, I've seen my suspicions about Matt M get reconfirmed multiple times. I saw him sue people and threaten them over small things, only to loose in court. I watched Gutenberg be a mess and witnessed more captains calls around it's release that upset people for no real reason.

Recently, he went on the offensive against WP Engine, which did not surprise me. If anything I was surprised that it happened now, not years ago.

Does he have a point? Yes, I agree that WPE could give back a lot more. I think that they should be part of the core team and be a top core contributor.

Did he mess up with how he approached it? Yes, I think it proves that he's too attached to run WordPress or be a big part of leadership. I thank him for everything he's done but please step down and let the community recover. Sure, contribute as much as you want, but let someone else make the calls. Advise them if need be, but the community's trust has been betrayed and it's time to step back.

Also, if you don't have clear, easy ways to let people mirror everything, then don't force it upon 200k+ websites. Attacking WPEngine isn't just attacking one person, it's attacking me as one of their thousands of users and that is why the betrayal hurts so much. 15 years of my life using WordPress and he's just pulled my core updates, plugin installs and other feature's over a spat with my hosting company. Sure it's been returned for the weekend, I just hope I don't loose more time to his emotions. Could you imagine the comments and headlines if a woman did this? I'd have to avoid the internet for weeks, the tech bros would be going off.

Will I contribute to Open Source in the future?

Hell yes, I've been using Laravel for 10 years now and they are the best community I've been apart of. It's never had the problems that WordPress had and continues to have.

I have started giving back lately with some small security based feedback to websites and packages. I even spent a little time on a open source project called LibreChat, I even coined the name when I threw it out as an option. Unfortunately I got a bit distracted with study for my OSCP last year, so I dropped off soon after, but I still use it daily and jump in if I see something wrong.

I have also learnt a lot of the Laravel core framework and have done a small contribution to Pest in recent times, all to help give back. I even started giving some small donations on GitHub, just what I can afford for now. Shout out to Saloon PHP, you have saved me so much time this past year.

I also hope to focus on improving security online, improving my own knowledge and possibly share anything neat I learn along the way.

Speaking of security, if you are after a neat security blog, check out https://securinglaravel.com/
Definitely relevant to the Laravel world, if thats the space you're in.
